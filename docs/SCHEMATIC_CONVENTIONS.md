# Schematic Conventions

Best practices для оформления схем в KiCad.

## Направление потоков

- Питание сверху, земля снизу
- Сигнал слева направо (вход → обработка → выход)
- Высокое напряжение слева, низкое справа (12V → 5V → 3.3V)

## Провода и соединения

- Провода только горизонтально/вертикально, никогда по диагонали
- Избегать 4-way junction (крест) — использовать T-образные
- Net labels для дальних соединений, провода для локальных
- Шины питания — через power symbols (GND, +5V), не провода через весь лист

## Компоненты

- Номиналы и reference видны, не перекрывают провода
- Bypass caps рядом с IC которые они обслуживают
- Поляризованные компоненты — «+» к питанию (вверх)
- Pin 1 / dot маркировка видна

## Организация

- Один функциональный блок на область (или отдельный лист)
- Title block заполнен (название, ревизия, дата)
- Текстовые аннотации для пояснения неочевидных решений
- Decoupling caps рядом с IC, не в отдельном углу

## Именование

- Reference designators последовательные, сгруппированы по функции
- Net names отражают функцию (VIN_SENSE), а не физику (PA1)
- Power nets — стандартные имена (+5V, +3V3, GND)

## Символы модулей (dev board и т.п.)

- Названия пинов — по даташиту платы (PA1, PB8), без привязки к домену проекта
- Назначение пина (VIN_SENSE, CAN_TX) определяется net label на схеме, а не в символе
